I’m saving to my Home Screen. Why are the nav icons showing?
The navigation icons (the back/forward arrows, share button, and compass) are appearing because your browser still considers this a standard web session rather than a "standalone" web app.

To hide that browser UI once the site is saved to the Home Screen, you need to configure the web app manifest or use specific meta tags to tell iOS that this is a Progressive Web App (PWA).

1. Add the "Web App Capable" Meta Tag
The quickest fix for iOS is to add this tag to the <head> of your HTML. This tells Safari to hide the URL bar and the bottom navigation toolbar when launched from the Home Screen:

HTML
<meta name="apple-mobile-web-app-capable" content="yes">
2. Configure the Web App Manifest
If you are using a manifest.json (which is best practice for GitHub Pages sites), ensure the display property is set to standalone or fullscreen:

JSON
{
  "name": "Pre-Flight Checklist",
  "display": "standalone",
  "start_url": "/index.html"
}
3. Key Troubleshooting Tips
The "New Window" Trap: If your Home Screen icon points to yoursite.github.io but then immediately redirects to a different URL (like a subfolder or a different domain), Safari will pop the navigation bar back out because it thinks you've "left" the app.

The Refresh Factor: Changes to the manifest or meta tags aren't retroactive. You’ll need to delete the existing icon from your Home Screen and "Add to Home Screen" again after pushing your code updates to GitHub.

Status Bar Styling: To make it look even more integrated, you can customize the top status bar (where the time and battery are) with this tag:

HTML
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
Once you add the apple-mobile-web-app-capable tag and re-save it, those bottom icons should vanish, giving you that full-screen "app" feel.

I do have an iPhone, but would like android to have the same experience
To ensure Android users get that same "standalone" app experience without the Chrome address bar or navigation buttons, you’ll need to lean on the Web App Manifest. While Apple uses specific meta tags, Android relies almost entirely on the manifest.json file.

1. The Manifest Essentials
For Chrome on Android to treat your site as an app, your manifest.json must include these specific fields:

JSON
{
  "short_name": "Checklist",
  "name": "Pre-Flight Checklist",
  "icons": [
    {
      "src": "icon-192.png",
      "type": "image/png",
      "sizes": "192x192"
    },
    {
      "src": "icon-512.png",
      "type": "image/png",
      "sizes": "512x512"
    }
  ],
  "start_url": "./index.html",
  "background_color": "#000000",
  "display": "standalone",
  "theme_color": "#000000"
}
display: standalone: This is the magic line. it removes the browser UI (address bar, navigation buttons) on Android.

theme_color: This matches the browser's address bar area to your app's background color, making the transition seamless.

2. Link the Manifest in HTML
Make sure your HTML <head> actually points to that file:

HTML
<link rel="manifest" href="/manifest.json">
<meta name="theme-color" content="#000000">
3. The Android "Install" Prompt
Unlike iOS, where users have to manually find "Add to Home Screen," Android can actually prompt the user to install your app. To make this happen, Chrome requires:

A Service Worker registered (even a simple, empty one will work for basic "standalone" mode).

The site must be served over HTTPS (which GitHub Pages does by default).

Comparison of Experience
Feature	iOS Implementation	Android Implementation
Hide Nav Icons	apple-mobile-web-app-capable tag	"display": "standalone" in manifest
Status Bar Color	apple-mobile-web-app-status-bar-style	"theme_color" in manifest
App Name	apple-mobile-web-app-title	"short_name" in manifest
Pro-Tip for Development
On Android, you can verify if your manifest is working by using Chrome DevTools. If you inspect your site, go to the Application tab and click Manifest. It will tell you exactly if any requirements are missing for the "Install" prompt or standalone mode to trigger.

Once you push these changes to GitHub, Android users who "Add to Home Screen" will see your checklist as a full-screen app, just like your iPhone experience.
